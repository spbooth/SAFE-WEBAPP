General configuration properties and logging

The following are general configuration parameters you might that may need to be set when creating a new application:

  • service.name This parameter identifies the name of the service supported by the application e.g. “ARCHER” . Keep this name short, ideally a single identifying name, as it is usually used to qualify other terms such as “web-site”, “service” “helpdesk” etc.

  • service.url The canonical URL of the application.

  • service.website-name The short name used to refer to the application itself.

  • mail.smtp.host Address of an SMTP server the application will use to send outgoing emails.

  • email.from_address The default FROM address for mails sent by the application

  • error.email_notify_address An email address where application internal errors should be reported by email

  • error.email_from_address The FROM address to use for error emails.

  • info.email_notify_address An email address where the application should send informational/warning messages.

  • info.email_from_address The FROM address to use for informational/warning messages.

  • email.force.address A debugging option that forces all emails from the application to be sent to only the specified email address.

  • service.systems Text describing the resources the application manages defaults to “service.name machines”





The application uses the Log4J logging package. Logging is configured via the application configuration system so you can add any standard Log4J logging configuration to the configuration files. For example to generate a rolling debug log in /tmp add:

logdir=${java.io.tmpdir}

# main log is rolled daily

log4j.logger.uk.ac=DEBUG, H

log4j.appender.H=org.apache.log4j.RollingFileAppender

log4j.appender.H.File=${logdir}/DebugSaf.log

log4j.appender.H.MaxFileSize=100KB

log4j.appender.H.MaxBackupIndex=100

log4j.appender.H.layout=org.apache.log4j.PatternLayout

log4j.appender.H.layout.ConversionPattern=%-5p %d %c %m %n




There is a special log location used to record resource allocation changes that you may want to capture explicitly:

# special resource logger rolled monthly

log4j.logger.uk.ac.hpcx.resource=DEBUG, X

log4j.appender.X=org.apache.log4j.DailyRollingFileAppender

log4j.appender.X.File=${logdir}/resource.log

log4j.appender.X.DatePattern=.yyyy-MM

log4j.appender.X.layout=org.apache.log4j.PatternLayout

log4j.appender.X.layout.ConversionPattern=%-5p %d %c %m %n
